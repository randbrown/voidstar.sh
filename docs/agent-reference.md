This document collects durable context and aesthetic direction for agents working on:

- **Randyland2**: the home/garden/art/consciousness space.
- **Ramblin’ Visioneer**: the stained-glass Sasquatch character/avatar for voidstar.sh and its apps.
- **Cindy Lynn’s Gardens / CindyLynn**: stained glass, flowers, science/chemistry, garden energy.
- **voidstar / voidstar.sh / qualia lab**: live coding, pedal steel, visual FX, ambient/avant-garde performance systems.
- Related album art, stickers, websites, shader work, audience participation, live visuals, and personal mythology.

Use this as a shared memory packet for coding agents, design agents, branding agents, visual artists, music agents, and writing agents so they can begin with the right vibe instead of starting from scratch.

### Bigfoot / Sasquatch direction

The Sasquatch should generally be:

- mystical
- friendly or smiling
- strange
- confident
- iconic
- a little shaggy
- a little cosmic
- not horror-monster
- not too cartoonishly goofy
- not too realistic

Preferred movement language for animation:

```text
Sasquatch Bigfoot swagger
```

Swagger walk details:

- wide stance
- loose arms
- hip-led motion
- slow confident weight shifts
- arms lag slightly behind the body
- feet may drag slightly
- shallow vertical bob
- micro phase drift to avoid robotic loops
- the figure can appear to walk while remaining centered on screen

### Bigfoot monster truck note

Bigfoot monster truck imagery may be referenced as an easter egg or energetic influence, but it should be **less prominent** than the mystical stained-glass Sasquatch guardian concept.

### animation vocabulary

When the Visioneer is animated (sprite sheet or rig), the canonical motion set is:

- `idle_breathe` — slow belly/chest/hair motion, cyan eye shimmer

## 7. qualia lab / voidstar.sh

### identity

**qualia lab** is the live visual FX / audio visualization platform associated with voidstar.sh.

It is a realtime browser-based performance system for:

- live coding
- pedal steel performance
- music-reactive visuals
- pose tracking
- audience interaction
- shader-based visual FX
- generative environments
- OBS / projection / recording workflows

It should feel like:

```text
a live visual instrument
```

Not a screensaver. Not a generic audio visualizer. Not a background loop. It is an expressive instrument that responds to sound, body, code, and performance decisions.

### core design principle

```text
qualia = consciousness rendered as light + sound rendered as geometry
```

The system should make invisible states visible:

- audio energy
- pose/motion
- audience presence
- code patterns
- breathing rhythms
- attention
- “observer” effects
- memory trails
- phase changes
- fields of influence

### target feel

Qualia should feel:

- cold, cosmic, intelligent
- digital but alive
- responsive but not chaotic
- ambient-performance ready
- capable of stillness
- capable of eruption
- readable at stage distance
- capable of long-form evolution
- shader-rich but performant

### technical context

Important implementation assumptions:

- browser-first
- client-only where possible
- GitLab Pages / static hosting compatible
- serverless preferred
- WebGL2 / canvas / Web Audio
- MediaPipe pose tracking has been used
- Strudel live coding may be involved
- OBS/projector workflows matter
- target performance around 30 fps or better
- visuals must not compromise live realtime performance
- pose/audio/param persistence handled by the broader app harness where relevant

### qualia plugin ideas / known directions

Known or desired FX include:

#### gargantua

Black-hole / Interstellar-inspired lensing environment.

Elements:

- accretion disk
- gravitational lensing
- polar jets
- bass controls brightness / mass / disk energy
- mids affect turbulence
- treble controls sparkles or particles
- camera orbit
- should be performant, aiming around 30 fps

#### voidstar_logo

A cosmic void portal with 3D or dimensional `void*` / voidstar text.

Elements:

- swirl
- portal
- logo reveal
- audio-reactive distortion
- pose/environment modulation
- useful for intros, title cards, and set transitions

#### synthwave

Retrowave/outrun grid and horizon spectrum.

Elements:

- grid pulses
- EQ horizon
- neon restraint
- not too generic
- should still feel voidstar, not stock synthwave

#### ghost in the machine

A ghostly / nebulous apparition forming above a digital circuit-board field.

Elements:

- particle blob
- audio-reactive breathing
- aura / ooze / Mysterio-like energy
- ghostly rim arcs
- particle trails
- apparition forms and dissolves
- circuit-board background
- should feel like a spirit emerging from computation

#### dog pose detection

Potential canine pose tracking experiments for visual FX involving dogs.

Important: dog pose detection should be treated as a specialized model/tooling problem, not assumed to work from human pose tracking.

#### observer field

Audience participation feature direction for qualia.

Possible names:

- Observer Field
- Qualia Entanglement
- The Observer Mesh
- Noösphere
- Event Horizon Choir
- Wavefunction Chorus
- Ruliad Relay
- Many-Minds Mode
- The Qualia Field

Core idea:

```text
audience members scan a QR code and become part of the live visual/audio system
```

Possible interactions:

- phone pose tracking
- audience-controlled visual parameters
- choosing qfx types
- tweaking selected parameters
- request/queue system
- remote sensor input
- collaborative field of influence
- possibly controlled pattern edits, carefully sandboxed

Constraints:

- must not harm live realtime performance
- should be serverless or static-host friendly if possible
- WebRTC / WebSocket-like patterns may be considered, but architecture must respect static hosting constraints
- performer remains in control
- audience input should be rate-limited, smoothed, sandboxed, and visually meaningful

#### vision / opencv layers

OpenCV-style processing can complement MediaPipe pose. Use it sparingly in-browser — avoid CPU-heavy full-resolution loops during a live set:

- optical flow → full-frame motion energy
- background subtraction → ghost trails
- contour fields → body silhouettes
- frame differencing → smear / glitch layers
- edge detection → CRT / vector overlays

### live aural coding (code as visual)

Treat code and sound as a shared score. The live-coding surface should read as part of the visual field, not an IDE screenshot:

- live code as a phosphor overlay, not a giant editor window
- Strudel pattern names as score annotations
- pattern density as "particle weather"
- parameter automation as visible "knob ghosts" or field lines
- code edits as localized wavefunction collapses
- syntax fragments as short-lived glyphs in the field
- tempo/cycle phase as orbital motion, not a metronome dot

A long-form set wants a visual arc, not constant intensity: quiet intro → slow evolution → stable middle → controlled eruption → return to stillness → a title/transition moment.

## 10. agent design rules

### for coding agents

When building voidstar / qualia features:

1. Treat visuals as performance instruments.
2. Preserve realtime performance and stable framerate.
3. Prefer graceful degradation over visual overload.
4. Make parameters live-tweakable.
5. Smooth noisy inputs.
6. Keep performer control central.
7. Make visual state legible from across a room.
8. Support recording/projector/OBS workflows.
9. Avoid hard dependencies on servers where static hosting is expected.
10. Keep the system modular: qfx plugins, audio analysis, pose input, UI controls, presets.

Useful technical patterns:

- `requestAnimationFrame` loop discipline
- WebGL2 fragment shaders for heavy visuals
- Web Audio feature extraction
- smoothed audio envelopes
- beat pulses as optional transient inputs
- pose landmark mapping to canvas coordinates
- parameter automation
- low-GC render loops
- static-host-compatible architecture
- optional local network or QR-based audience layer, carefully sandboxed

### for visual design agents

When designing for Randyland / voidstar:

1. Fuse organic, handmade, and cosmic/computational imagery.
2. Use stained glass as a structural motif, not just a texture.
3. Use Sasquatch as mythic guide/trickster/guardian.
4. Treat Mahakala / angels / Evil Eye as protective symbolic anchors.
5. Keep voidstar more minimal, dark, and code-cosmic.
6. Keep Cindy Lynn’s garden/science branch luminous, floral, and chemistry-aware.
7. Let performance visuals breathe.
8. Make designs work in square, vertical phone, sticker, and stage-projection formats.

## 13. dos and don’ts

### do

- Keep the personal mythology intact.
- Use `@randbrown` as the canonical personal handle.
- Keep `voidstar` lowercase unless there is a specific reason not to.
- Treat `voidstar.sh` as a live performance/software platform.
- Treat `qualia lab` as a visual instrument, not just an app.

- Design for stickers, decals, square covers, vertical phone screens, and live projection.
- Make coding output modular, performant, and performance-safe.

### don’t

- Don’t flatten everything into generic “psychedelic art.”
- Don’t make voidstar look like stock cyberpunk/EDM visuals.
- Don’t make Ramblin’ Visioneer too horror, too cute, or too realistic.

- Don’t sacrifice live-performance stability for flashy effects.
